¡Gran decisión! RustDesk es una alternativa excelente a Chrome Remote Desktop y TeamViewer: es de código abierto, infinitamente más rápida de configurar en Linux y no suele dar los dolores de cabeza de las pantallas negras.
Como Kali Linux está basado en Debian, la instalación es sumamente sencilla usando el paquete oficial .deb. Aquí tienes el paso a paso:
Guía de Instalación de RustDesk en Kali Linux
Paso 1: Actualizar el sistema
Como buena práctica en Kali, abre una terminal y actualiza la lista de paquetes para evitar conflictos de dependencias:
sudo apt update

Paso 2: Descargar el paquete oficial .deb
Puedes descargar la versión para sistemas de 64 bits (AMD64/x86_64) directamente desde el GitHub oficial con este comando wget:
wget https://github.com/rustdesk/rustdesk/releases/download/1.4.8/rustdesk-1.4.8-x86_64.deb

(Nota: Si al ejecutarlo ves que salió una versión más reciente, puedes descargarla directamente desde su página oficial de descargas).
Paso 3: Instalar el paquete con APT
Para asegurarnos de que el sistema instale automáticamente cualquier librería o dependencia que le haga falta, utiliza apt install apuntando al archivo que acabas de descargar:
sudo apt install ./rustdesk-1.4.8-x86_64.deb -y

> 💡 ¿Por qué apt en lugar de dpkg? Si usas dpkg -i, es muy probable que te dé errores de dependencias rotas. Usar apt install ./nombre_archivo.deb resuelve y descarga todo lo que falte automáticamente.
> 
Paso 4: ¡Listo! Abrir RustDesk
Una vez que termine la instalación, ya puedes buscar RustDesk en el menú de aplicaciones de tu Kali Linux (bajo la categoría de herramientas normales o simplemente escribiendo su nombre), o bien lanzarlo directamente desde la terminal corriendo:
rustdesk &

Al abrirlo, verás inmediatamente tu ID y una Contraseña aleatoria en el panel izquierdo. Ya puedes usar esos datos para conectarte a tu Kali desde cualquier otro dispositivo con RustDesk instalado.
🛠️ Un par de tips para Kali Linux:
 * ¿Estás usando el usuario root? A diferencia de Chrome Remote Desktop, RustDesk suele abrirse sin problemas bajo root. Sin embargo, si por alguna razón no arranca la interfaz gráfica, asegúrate de correrlo desde tu usuario normal (kali).
 * Inicio automático: Si quieres que RustDesk se inicie solo cada vez que enciendas tu máquina Kali (ideal si la tienes en un servidor o máquina virtual sin monitor), el propio instalador ya configura el servicio de fondo. Puedes verificar que esté corriendo con:
   sudo systemctl status rustdesk

Configurar Chrome Remote Desktop en Kali Linux es una excelente opción para acceder a tu entorno de pruebas de forma remota. Dado que Kali está basado en Debian y utiliza XFCE como entorno de escritorio predeterminado, el proceso es bastante directo, pero requiere un par de trucos de configuración para evitar la famosa "pantalla negra".
> ⚠️ Nota importante de seguridad: Chrome Remote Desktop no se lleva bien con el usuario root por motivos de seguridad de Chromium. Asegúrate de hacer todo este proceso desde un usuario normal (como el usuario por defecto kali).
> 
Guía Paso a Paso
Paso 1: Actualizar el sistema
Antes de instalar cualquier paquete, abre tu terminal y asegúrate de tener los repositorios al día:
sudo apt update && sudo apt upgrade -y

Paso 2: Instalar Google Chrome
Chrome Remote Desktop requiere que el navegador base esté instalado en el sistema. Si ya lo tienes, puedes saltar al Paso 3.
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install ./google-chrome-stable_current_amd64.deb -y

Paso 3: Descargar e instalar Chrome Remote Desktop
Descarga el paquete oficial .deb del host de escritorio remoto e instálalo:
wget https://dl.google.com/linux/direct/chrome-remote-desktop_current_amd64.deb
sudo apt install ./chrome-remote-desktop_current_amd64.deb -y

Si la terminal te arroja algún error de dependencias, soluciónalo rápidamente corriendo: sudo apt --fix-broken install
Paso 4: Añadir tu usuario al grupo correcto
Para que el servicio pueda ejecutarse bajo tu cuenta sin pedir permisos de administrador constantemente, añade tu usuario al grupo de Chrome Remote Desktop:
sudo usermod -aG chrome-remote-desktop $USER

Paso 5: Configurar el entorno de escritorio (Evita la pantalla negra)
Chrome Remote Desktop necesita saber exactamente qué interfaz gráfica debe iniciar cuando te conectes. Como Kali usa XFCE, crea un archivo de configuración en tu directorio personal con este comando:
echo "exec /usr/bin/xfce4-session" > ~/.chrome-remote-desktop-session

Paso 6: Generar el código de vinculación de Google
Ahora necesitas autorizar a tu máquina Kali para que se conecte a los servidores de Google:
 * Desde otra computadora (o desde el propio Chrome en Kali), abre el navegador e ingresa a: remotedesktop.google.com/headless.
 * Inicia sesión con tu cuenta de Google.
 * Haz clic en Configurar otra computadora y avanza hasta que te muestre un comando largo para Debian Linux.
 * Haz clic en el botón de Copiar (copiará un comando que empieza con DISPLAY= /opt/google/chrome-remote-desktop/start-host...).
 * Pega ese comando en la terminal de tu Kali Linux y presiona Enter.
 * El sistema te pedirá que crees un PIN de 6 dígitos. Elígelo bien y confírmalo (no verás los números mientras los escribes por seguridad).
Paso 7: Iniciar el servicio
Para asegurar que todo empiece a correr con la nueva configuración, reinicia el servicio del escritorio remoto:
sudo systemctl restart chrome-remote-desktop@$USER

¿Cómo conectarte ahora?
¡Listo! Tu Kali Linux ya está en la nube de Google.
 * Ve a remotedesktop.google.com/access desde cualquier dispositivo (PC, Mac, Android o iOS).
 * Verás el nombre de tu máquina Kali en la lista de dispositivos "En línea".
 * Haz clic en ella, introduce el PIN de 6 dígitos que creaste en el Paso 6 y ¡estarás dentro!
> 💡 Tip Pro: Si al conectarte de forma remota ves una pantalla negra o se congela, suele ser porque tienes la sesión gráfica abierta físicamente en la máquina Kali al mismo tiempo. Prueba cerrar sesión (Log out) en tu máquina física y luego intenta conectarte desde el dispositivo remoto.
